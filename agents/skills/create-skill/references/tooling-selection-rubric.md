# Tooling Selection Rubric

Use this rubric to rank CLI/library candidates before asking the user to choose.

## Scoring dimensions (1-5 each)
1. Task relevance: How directly does it solve the required workflow?
2. Documentation quality: Are examples and setup docs clear and current?
3. Ecosystem maturity: Is it maintained and widely used?
4. Operational risk: How likely is breakage or unsafe behavior?
5. Adoption cost: How heavy are install, runtime, and integration steps?

## Suggested weighting
- Task relevance: 30%
- Documentation quality: 25%
- Ecosystem maturity: 20%
- Operational risk: 15%
- Adoption cost: 10%

## Output format
Provide users with:
- Top 2-4 ranked options
- One-line recommendation
- Tradeoff summary per option
- Direct question asking which option to adopt

## Interview tie-breakers
If scores are close, ask:
- Prefer minimal dependencies or richer feature set?
- Prefer quickest setup or best long-term maintainability?
- Must align with existing team standards?
